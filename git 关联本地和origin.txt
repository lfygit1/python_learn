1.公司有仓库的话直接在本地路径下cmd执行git clone 仓库地址
2.本地开发了，但是远程没有仓库
在远程建立新仓库，（名字必须与本地路径下名字一致）
本地依次输入以下命令
git init      # 创建 .git/ 目录并初始化基本的 Git 配置和对象数据库
git add .   # 把当前目录及其子目录中发生的“新文件”和“修改过的文件”加入到暂存区（staging area）
git commit -m "first commit"     # 把当前暂存区（staged）的改动打包成一个新的提交（commit）
git branch -M main      # 把当前所在分支重命名为 main。-M 是强制重命名（--move --force），如果 main 已存在则覆盖（强制移动/替换）
git branch        # 列出本地分支，当前分支前有一个 *
git status        # 也显示你所在的分支
git remote add origin <远程仓库地址>
git push -u origin main    # 把本地 main 分支的提交推送到远程仓库 origin 并创建/更新远程的同名分支；-u（或 --set-upstream）表示同时把本地分支与远程分支建立“上游（tracking）”关系，之后可以直接用 git push / git pull（不用再写 origin main）
