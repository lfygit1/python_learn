一、Fiddler 简介

Fiddler 是一款功能强大的 HTTP 调试代理工具，能够记录客户端和服务器之间的所有 HTTP/HTTPS 通信。它通过代理方式工作，允许您检查、修改和重发请求，是 Web 开发、测试和安全分析的必备工具。

流量监控

实时监控所有 HTTP/HTTPS 请求和响应，包括请求头、响应头、请求体和响应体。

断点调试

在请求发送前或响应返回前设置断点，修改请求或响应内容，测试不同场景。

性能分析

分析请求时间线，识别性能瓶颈，优化网站或应用加载速度。

安全测试

检查传输中的数据安全，测试 API 接口安全性，发现潜在的安全漏洞。

三、所有可用功能介绍

核心功能

  • 流量捕获：捕获所有经过代理的 HTTP/HTTPS 请求
  • 请求/响应查看：以多种格式（原始、头部、JSON、XML 等）查看请求和响应内容
  • 断点设置：全局断点或针对特定请求设置断点
  • 请求篡改：修改请求参数、头部、请求体等
  • 响应篡改：修改服务器返回的响应内容
  • 自动响应：创建规则，自动用本地文件或预设响应替换服务器响应
  • 过滤器：根据主机、状态码、请求类型等过滤显示的请求
  • 性能统计：统计请求时间、大小、缓存命中率等
  • 时间线视图：图形化显示请求的时间线，方便性能分析

高级功能

  • 弱网模拟：模拟低速网络环境
  • 断言：对响应内容进行断言测试
  • 脚本扩展：使用 FiddlerScript 扩展功能
  • API 测试：构建和发送自定义 HTTP 请求
  • 会话比较：比较两个会话的差异
  • 编码/解码工具：内置多种编码解码工具

四、监听电脑端操作方法

基本设置

启动 Fiddler，确保左下角 "Capturing" 按钮为开启状态（或按 F12 键）
打开浏览器，访问任意网站
观察 Fiddler 主界面左侧的会话列表，应该能看到捕获的请求

配置浏览器代理

大多数情况下，Fiddler 会自动设置系统代理。如果遇到问题，可以手动配置：

打开浏览器设置，找到代理设置
设置代理服务器地址为 127.0.0.1（本地主机）
设置代理端口为 8888（Fiddler 默认端口）
保存设置并刷新页面
提示：Fiddler 默认只捕获进程外流量（如浏览器）。要捕获进程内流量（如 .NET 应用程序），需要在应用代码中配置代理。

五、监听手机端操作方法

前提条件

  • 手机和电脑必须在同一局域网（连接同一 Wi-Fi）
  • 知道电脑的局域网 IP 地址（在 Fiddler 中可查看）

配置步骤

在 Fiddler 中，打开菜单 Tools > Options > Connections
勾选 "Allow remote computers to connect"
记住 Fiddler 监听端口（默认 8888）
重启 Fiddler 使设置生效
在手机上，进入 Wi-Fi 设置，修改当前连接的网络
设置代理为手动，输入电脑的 IP 地址和端口号（8888）
在手机浏览器中访问 http://电脑IP:8888
下载并安装 Fiddler 根证书
完成设置，手机上的所有 HTTP/HTTPS 流量都将经过 Fiddler
重要：测试结束后，请记得关闭手机上的代理设置，否则手机可能无法正常上网。

六、监听 HTTP 的操作方法

HTTP 监听是 Fiddler 的默认功能，无需额外配置：

确保 Fiddler 正在运行且捕获功能已开启
进行任何 HTTP 请求（访问网站、使用应用等）
观察 Fiddler 会话列表，所有 HTTP 请求都会显示
点击任意会话，查看详细信息

查看请求详情

右侧面板提供多个标签页查看请求和响应：

  • Headers：查看请求和响应头部
  • TextView：查看原始文本内容
  • SyntaxView：格式化显示 JSON、XML 等结构化数据
  • WebForms：查看表单提交的参数
  • HexView：十六进制视图
  • Auth：认证信息
  • Caching：缓存相关信息
  • Raw：原始请求/响应数据

七、监听 HTTPS 的操作方法

HTTPS 解密配置

在 Fiddler 中，打开菜单 Tools > Options > HTTPS
勾选 "Decrypt HTTPS traffic"
在弹出的证书安装提示中，点击 "Yes"
选择证书存储位置（建议选择"受信任的根证书颁发机构"）
点击 "OK" 保存设置
重启 Fiddler

验证 HTTPS 解密

访问一个 HTTPS 网站（如 https://www.google.com）
在 Fiddler 会话列表中，HTTPS 请求应显示为明文
点击会话，查看解密后的请求和响应内容
安全提示：Fiddler 的 HTTPS 解密是通过中间人攻击方式实现的。在非测试环境中使用后，建议删除 Fiddler 根证书。

八、其他高级功能使用方法

弱网测试

模拟低速网络环境，测试应用在不良网络条件下的表现：

打开菜单 Rules > Performance > Simulate Modem Speeds
或者使用更精确的控制：Rules > Customize Rules
在打开的脚本文件中，找到 OnBeforeRequest 函数
添加延迟代码，例如：oSession["request-trickle-delay"] = "300";（单位：毫秒/字节）
保存脚本，Fiddler 会自动应用新规则

断言功能

对响应内容进行自动化断言测试：

选择要测试的会话
右键点击，选择 Save > Response > Response Body 保存响应
打开菜单 Rules > Customize Rules
在 OnBeforeResponse 函数中添加断言代码
示例代码：检查响应是否包含特定内容
static function OnBeforeResponse(oSession: Session) {
    if (oSession.uriContains("example.com")) {
        var body = oSession.GetResponseBodyAsString();
        if (!body.Contains("expected_text")) {
            oSession["ui-color"] = "red";
            oSession["ui-bold"] = "true";
        }
    }
}

自动响应（AutoResponder）

创建规则，自动用本地文件或预设响应替换服务器响应：

点击右侧 "AutoResponder" 标签
勾选 "Enable rules" 和 "Unmatched requests passthrough"
点击 "Add Rule" 按钮创建新规则
设置匹配规则（如 URL 包含特定字符串）
设置响应动作（如返回本地文件、重定向等）
保存规则并启用
